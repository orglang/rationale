# Что такое трансакция

> — Видишь суслика?  
> — Нет.  
> — И я не вижу. А он есть!

> [!CAUTION]
> Несмотря на то, что `транзакция` является устоявшимся понятием в IT-индустрии, мы будем использовать очень схожий термин `трансакция` в абсолютно другом значении.

Начнем с примера, в котором точно нет трансакций.

```go
func DoSomeWork() WorkOutcome {
    outcome := DoFirstTask()
    return DoSecondTask(outcome)
}
```
<p align="center"><sub>Листинг 1. Чистая работа</sub></p>

Считаем, что `DoFirstTask` и `DoSecondTask` являются чистыми функциями без побочных эффектов.

> [!NOTE]
> Нет побочных эффектов => трансакций тоже нет

Рассмотрим следующий пример.

```go
func DoSomeWork() WorkOutcome {
    provider := NewWorkProvider()
    outcome := provider.DoFirstTask()
    return provider.DoSecondTask(outcome)
}
```
<p align="center"><sub>Листинг 2. Работа через провайдера</sub></p>

В примере из листинга 1 функция `DoSomeWork` тоже была чистой. Теперь же мы в рамках нее создается провайдер, которому делегируется выполнение работы. Данная операция является побочным эффектом, т.к. при её выполнении аллоцируются вычислительные ресурсы. Можно ли её назвать трансакцией?
